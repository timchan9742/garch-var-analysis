# Estimating Portfolio Value-at-Risk with GARCH and EGARCH Models

## Overview

This project investigates the market risk of an equally weighted multi-asset portfolio using volatility models from the GARCH family. The objective is to estimate and compare one-day-ahead 95% Value-at-Risk (VaR) forecasts generated by GARCH(1,1) and EGARCH(1,1) models.

The analysis includes:

- Exploratory Data Analysis (EDA)
- Volatility modeling
- Model diagnostics
- Rolling out-of-sample forecasting
- VaR backtesting and comparison

---

## Portfolio Composition

The portfolio consists of five equally weighted assets (20% each):

| Asset Class | Instrument |
|------------|------------|
| Equity Index | S&P 500 (^GSPC) |
| Stock | NVIDIA (NVDA) |
| Currency Pair | EUR/USD (EURUSD=X) |
| Commodity | Gold (GC=F) |
| Cryptocurrency | Ethereum (ETH-USD) |

Daily adjusted prices are obtained from Yahoo Finance and transformed into log returns.

## Methodology

### 1. Data Preparation

- Download historical market data
- Align trading calendars across assets
- Forward-fill missing observations
- Calculate daily log returns
- Construct an equally weighted portfolio

### 2. Exploratory Data Analysis

The notebook investigates common stylized facts of financial returns:

- Volatility clustering
- Heavy-tailed distributions (leptokurtosis)
- ARCH effects
- Weak autocorrelation of returns

Diagnostic tools include:

- Histograms
- Q-Q plots
- ACF/PACF plots
- Ljung–Box tests
- ARCH-LM tests

### 3. Volatility Modeling

Two volatility specifications are estimated using Student-t innovations:

#### GARCH(1,1)

Captures time-varying volatility through conditional variance dynamics.

#### EGARCH(1,1)

Extends GARCH by allowing asymmetric responses to positive and negative shocks.

Model selection is based on:

- Bayesian Information Criterion (BIC)
- Residual diagnostics
- ARCH effect elimination

### 4. Model Diagnostics

The following diagnostic tests are performed:

- Ljung–Box tests on standardized residuals
- Ljung–Box tests on squared standardized residuals
- ARCH-LM tests
- Residual ACF diagnostics
- Parameter stability tests
- Engle–Ng sign bias tests
- Goodness-of-fit evaluation

### 5. Value-at-Risk Estimation

The project computes one-day 95% VaR using a volatility-adjusted historical simulation framework.

Both models generate:

- In-sample VaR estimates
- Out-of-sample rolling VaR forecasts

### 6. Walk-Forward Forecasting

A rolling expanding-window approach is used:

1. Re-estimate the model using all data available up to day *t − 1*
2. Forecast next-day volatility
3. Compute one-day-ahead VaR
4. Compare forecasted VaR with realized returns
5. Repeat through the entire out-of-sample period

### 7. VaR Backtesting

Forecast accuracy is evaluated using:

- Number of VaR breaches
- Breach rate
- Kupiec Unconditional Coverage Test
- Average exceedance beyond VaR

---

## Key Findings

- Portfolio returns exhibit heavy tails and volatility clustering.
- Both GARCH(1,1) and EGARCH(1,1) successfully capture conditional heteroskedasticity.
- The EGARCH asymmetry parameter is close to zero, suggesting limited leverage effects.
- Both models pass standard VaR backtesting procedures.
- Forecast performance is highly similar across specifications.
- The simpler GARCH(1,1) model provides comparable predictive performance and may be preferred for its interpretability and parsimony.
